# Foundry SDK を使用した Contoso Travel（プロンプトエージェント）の構築

この時点で、_インフラのセットアップ_ と _開発環境_ のセットアップが完了しており、Azure サブスクリプション内の既存の Foundry プロジェクトに接続された GitHub Codespaces が起動しているはずです。

ノートブックを順番に実行し、Contoso Travel エージェントを構築し、カスタムデータとツールでグラウンディングし、評価とトレースを行い、AI レッドチーミングスキャンを実行して攻撃への脆弱性を評価する過程を歩みます。

## 方法: ノートブックの実行

GitHub Codespaces 環境には、Jupyter ノートブックをサポートするための拡張機能とツールがプリインストールされています。ノートブックを「実行」するには、次のアクションを実行してください：

1. Visual Studio Code でノートブック（`.ipynb` ファイル）を開きます。
1. **カーネルを選択** をクリックし、**Python 3.12** オプションを選択します。
1. まずすべての出力をクリアしてから、**「すべて実行」ではなくセルを 1 つずつ実行** します。
1. 次に進む前に、各ステップの結果を観察して理解してください。

**重要**: ラボが完了してコードを理解するまで、ノートブックの最後のセルを実行しないでください。最後のセルを実行すると、ノートブックの出力だけでなく、Foundry で作成された関連アセット（例: エージェント）もクリアされます。
